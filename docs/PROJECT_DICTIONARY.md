## `packages/core/` — The Brain

> Pure TypeScript. Zero external dependencies. No React, no HTTP — just logic.
> Must be compiled (`npm run build -w @orgblueprint/core`) before `apps/web` can use it.

| File | What it does |
|---|---|
| `src/types.ts` | **Every shared TypeScript type in the whole project** — `BlueprintResult`, `Signals`, `ProductKey`, `ClarificationAnswers`, etc. If you want to know what shape the data is, start here. |
| `src/rules.ts` | **The core engine.** `extractSignals(text)` reads the user description → `decideProducts()` picks Salesforce products → `generateBlueprint()` assembles the full result. Also contains `parseUsers()` for detecting headcount from natural language. |
| `src/templates.ts` | `enrichWithTemplates(result, signals)` — overlays polished narrative text onto rules engine output for Demo mode. Covers: executive summary, why mapping, objects, analytics, roadmap, document checklist, risks. |
| `src/estimateLicenses.ts` | Calculates directional Salesforce license cost ranges based on products selected and user count |
| `src/estimateImplementation.ts` | Calculates directional implementation cost ranges (SI partner hours, complexity) |
| `src/pricingAssumptions.ts` | Raw pricing data and assumptions used by the two estimators above |
| `src/index.ts` | Barrel export — re-exports everything so `apps/web` can import from `@orgblueprint/core` |
| `test/regression.test.ts` | Tests that the rules engine always picks correct products for known inputs |
| `test/users.test.ts` | Tests that user count parsing works for all edge cases (role names, summing multiple groups) |
| `tsconfig.json` | TypeScript config for the core package |
| `dist/` | Compiled output — generated by build command. Never edit manually. |

### `src/auth.ts` — Authentication Config

NextAuth v5 setup. Defines the credentials provider (email + bcrypt password), JWT session strategy, and session callbacks. `trustHost: true` is required so CSRF validation doesn't block local dev.

## Two Packages — How They Connect

```
packages/core  (rules engine, types, cost estimators)
      ↓  compiled to dist/ via tsc
apps/web       (Next.js app imports @orgblueprint/core)
      ↓  API routes call core functions
      ↓  UI components render the results
Browser        (user sees the blueprint)
```

**Rule:** always rebuild core after changing it — `npm run build -w @orgblueprint/core` — otherwise the web app uses the old compiled version.

---

## Three Modes — How They Work

| Mode | What runs | Speed | Requires |
|---|---|---|---|
| **Demo** | `rules.ts` + `templates.ts` only | Instant | Nothing — no API keys needed |
| **AI Enhanced** | Orb asks up to 5 questions via `/api/conversation`, then LLM generates blueprint via `/api/blueprint` | 5–20 seconds | At least one AI API key |
| **Chat widget** | `/api/chat` answers questions about the current blueprint | Per message | At least one AI API key |

**AI fallback chain:** Anthropic Claude Sonnet → Gemini 2.0 Flash → Groq → deterministic rules engine

## Where Do I Look If…

| Situation | File to open |
|---|---|
| Blueprint output is wrong | `packages/core/src/rules.ts` |
| Demo mode text looks bad | `packages/core/src/templates.ts` |
| AI questions are repeating or irrelevant | `apps/web/src/lib/anthropic.ts` → `buildQuestionPrompt()` |
| Cost estimate is off | `packages/core/src/estimateLicenses.ts` |
| Sign in / sign out broken | `apps/web/src/auth.ts` |
| Database schema change needed | `apps/web/prisma/schema.prisma` then `npx prisma db push` |
| Add a new page | `apps/web/src/app/your-page/page.tsx` |
| Add a new API endpoint | `apps/web/src/app/api/your-endpoint/route.ts` |
| Wizard flow or stages | `apps/web/src/components/ConversationChat.tsx` |
| Dashboard tabs or sections | `apps/web/src/components/BlueprintDashboard.tsx` |
| Floating chat widget | `apps/web/src/components/AIAssistantWidget.tsx` |
| PDF export broken | `apps/web/src/lib/exportPdf.ts` |
| AI quota / rate limiting | `apps/web/src/lib/quota.ts` |
| User count not detecting | `packages/core/src/rules.ts` → `parseUsers()` |
| AppExchange recommendations wrong | `apps/web/src/lib/appExchange.ts` |
| License pricing data wrong | `apps/web/src/lib/pricing.ts` |
| Technical tab content wrong | `apps/web/src/lib/technicalBlueprint.ts` |
| Roadmap checklist items wrong | `apps/web/src/lib/implementationChecklist.ts` |
| Run E2E tests | `cd apps/web && npm run test:e2e` |
| Run rules engine tests | `npm run test:core` |
| Dev server not starting | Check PM2: `pm2 status` then `pm2 restart orgblueprint-3000` |
